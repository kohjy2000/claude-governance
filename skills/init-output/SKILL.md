# /init-output — Output type 초기화

`$ARGUMENTS`: `figures` | `writing` | `grant`. 하나만 지정.

**Schema reference**: `~/.claude/blueprints/schemas/OUTPUTS.schema.md`.

---

## Step 1: Precondition

1. 현재 디렉토리가 프로젝트 루트인지 확인:
   ```bash
   test -f CLAUDE.md && test -d docs || { echo "프로젝트 루트가 아님. CLAUDE.md + docs/ 필요."; exit 1; }
   ```
2. `$ARGUMENTS` 검증: `figures`, `writing`, `grant` 중 하나인지. 아니면 STOP.
3. 이미 `outputs/<type>/` 존재 → "이미 초기화됨" 보고 후 STOP. 덮어쓰기 금지.

---

## Step 2: Type별 생성

### 2-1. figures
```bash
mkdir -p outputs/figures/code outputs/figures/panels
touch outputs/figures/FIGURE_PLAN.md
touch outputs/figures/PANEL_REGISTRY.md
touch outputs/figures/REVIEW_LOG.md
```

Placeholder 내용:
- `FIGURE_PLAN.md`: 
  ```
  # Figure Plan
  > Generated by /init-output. Populate via /figure-plan.
  > Schema: ~/.claude/blueprints/schemas/FIGURE_PLAN.schema.md
  ```
- `PANEL_REGISTRY.md`:
  ```
  # Panel Registry
  > Generated by /init-output. Updated by /figure-implement.
  > Schema: ~/.claude/blueprints/schemas/PANEL_REGISTRY.schema.md

  | Panel | Variant | File | Format | Status | Selected at | Notes |
  |-------|---------|------|--------|--------|-------------|-------|
  ```
- `REVIEW_LOG.md`:
  ```
  # Figure Review Log
  > Append-only. Managed by /figure-review subagent + hook escalation (Phase 6).
  > Schema: ~/.claude/blueprints/schemas/REVIEW_LOG.schema.md
  ```

### 2-2. writing

mkdir -p outputs/writing/drafts
touch outputs/writing/DRAFT_LOG.md
```

Placeholder:

  ```

- `REVIEW_LOG.md`:
  ```

  ```

## Step 3: CLAUDE.md 업데이트 제안

프로젝트 `CLAUDE.md`에 outputs 섹션이 없으면 user에게 다음 내용 추가 제안 (AskUserQuestion):

```markdown
## Output Pipeline
- Path: `outputs/<type>/`
- Types active: <figures | writing | grant> (업데이트 시점 기준)
- Schema: `~/.claude/blueprints/schemas/OUTPUTS.schema.md`
```

User 승인 후에만 CLAUDE.md edit.

---

## Step 4: README.md 업데이트 제안

프로젝트 `docs/README.md`의 Status 섹션에 "Outputs initialized: <type>" 한 줄 추가 제안. User 승인 필수.

---

## Step 5: 요약 출력

```
--- Output initialized ---
Type: <figures | writing | grant>
Path: outputs/<type>/
Files created: <list>
Directories: <list>

다음 단계:
- figures: /figure-plan 으로 design doc 작성 시작
- writing: outputs/writing/drafts/에 draft 파일 생성
- grant: outputs/grant/AIMS.md 편집
---
```

---

## Idempotency

- 같은 type 재호출 → Step 1-3에서 "이미 존재" STOP.
- 다른 type 호출은 정상 진행 (셋 다 병렬 가능).

---

## Common Pitfalls

| Pitfall | Fix |
|---------|-----|
| docs/ 없는 위치에서 호출 | Step 1 precondition fail |
| `$ARGUMENTS`에 여러 type (`figures writing`) | 분리해서 각각 호출 |
| 기존 outputs/<type>/ 건드리기 | 덮어쓰기 금지 — Step 1에서 STOP |
| Writing 디렉토리에 schema 없다고 figure-plan 호출 | 다른 스킬 호출은 이 스킬 out of scope |

---

## Handoff
- figures 초기화 후 → `/figure-plan` (Phase 1 skill)
- writing 초기화 후 → draft 작성은 수동 또는 Phase 5 이후 draft-helper skill (미정)
- grant 초기화 후 → AIMS.md 수동 편집 또는 Phase 5 이후 grant skill (미정)
